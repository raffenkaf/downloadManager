пример использования 

$pathFrom = "https://pp.vk.me/c620219/v620219421/1ec/u5Dmz82svW8.jpg";
$pathTo = "/tes";

$mng = new ImgManager();
$mng->downloadImg($pathTo, $pathFrom, true);

Описание работы:

Основной метод - $mng->downloadImg();

принимает 4 аргумента

1 аргумент - внутренний путь назначения, указывать нужно без формата, но с названием файла
2 аргумент - внешний путь источника файла, есть 2 варианта указания(зависит от 3 аргумента) либо полный урл как в примере выше, 
             либо название ftp без префикса (например - полный урл "ftp://ftp.funet.fi/pub/standards", 
             в аргумент нужно передавать "ftp.funet.fi/pub/standards" ). Нужно передавать путь до изображения. 
3 аргумент - флаг, который указывает нужно брать полный урл или частный случай(ftp), принимает булевый параметр, 
             по умолчанию false(частный случай(ftp))
4 аргумент - размер файла(байты), если задан то идет проверка размера файла, по умолчанию null

Метод может выбросить Exception, поэтому разумно окружить try catch блоком

При создании обьекта можно 3 аргумента

$mng = new ImgManager($ftp=null, $username=null, $password=null);

1 аргумент - путь фтп соединения, по умолчанию null(задается по тому же принципу что и в downloadImg)
2 аргумент - username для фтп соединения, по умолчанию null
3 аргумент - пароль для фтп соединения, по умолчанию null

Так же есть доп методы для задания параметров фтп соединения

$mng->setFtp(); - путь фтп соединения(задается по тому же принципу что и 2 аргумент в downloadImg())
$mng->setUsernameAndPassword(); - соответственно Username и Password
